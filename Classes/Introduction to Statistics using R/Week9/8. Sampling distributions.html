Sampling distributions¶

In this lab we will

  • take some random samples and
  • build sampling distributions
  • investigate the ways in which the statistics from a random sample of data can serve as point estimates for population parameters.
  • formulate a sampling distribution of our estimate in order to learn about the properties of the estimate, such as its distribution.

First, set a seed at the start of your lab.

A note on setting a seed:
  • Setting a seed will cause R to select the same sample each time you knit your document.
  • This will make sure your results don’t change each time you knit, and it will also ensure reproducibility of your work (by setting the same seed it will be possible to reproduce your results).
  • You can set a seed like this:
    • set.seed(35797) # make sure to change the seed
    • The number above is completely arbitraty. If you need inspiration, you can use your ID, birthday, or just a random string of numbers.
    • The important thing is that you use each seed only once in a document.
    In [1]:
    set.seed(20200512)

    The sampling distribution¶

    • gathering information on an entire population is often extremely costly or impossible.
    • take a sample of the population and use that to understand the properties of the population.

    If you are interested in estimating the proportion of people who don't think the work scientists do benefits them,

    • use the sample_n function to survey the population.
    In [8]:
    samp1 <- global_monitor %>%
  sample_n(50)

    • collects a simple random sample of size 50 from the global_monitor dataset, and

    • assigns the result to samp1.

    similar to randomly drawing names from a hat that contains the names of all in the population.

    Working with these 50 names is considerably simpler than working with all 100,000 people in the population.

    Suppose we took two more samples, one of size 100 and one of size 1000. Which would you think would provide a more accurate estimate of the population proportion?

  • every time you take another random sample, you might get a different sample proportion.
  • the distribution of sample proportions, called the sampling distribution (of the proportion), can help you understand how much variability you should expect when estimating the population mean.
    • The sampling distribution of the sample proportion
  • take 15,000 different samples of size 50 from the population,
  • calculate the proportion of responses in each sample,
  • filter for only the Doesn't benefit responses, and
  • store each result in a vector called sample_props50
  • visualize the distribution of these proportions with a histogram.
    • In [13]:
    sample_props50 <- global_monitor %>%
                    rep_sample_n(size = 50, reps = 15000, replace = TRUE) %>%
                    count(scientist_work) %>%
                    mutate(p_hat = n /sum(n)) %>%
                    filter(scientist_work == "Doesn't benefit")
    In [14]:
    ggplot(data = sample_props50, aes(x = p_hat)) +
  geom_histogram(binwidth = 0.02) +
  labs(
    x = "p_hat (Doesn't benefit)",
    title = "Sampling distribution of p_hat",
    subtitle = "Sample size = 50, Number of samples = 15000"
  )

    Note that we specify that replace = TRUE since sampling distributions are constructed by sampling with replacement.

    Sample size and the sampling distribution¶

    the sampling distribution of the proportions from samples of 50 people.¶

    In [18]:
    ggplot(data = sample_props50, aes(x = p_hat)) +
  geom_histogram(binwidth = 0.02)
    • the sampling distribution of the sample proportion is centered at the true population proportion, and
    • the spread of the distribution indicates how much variability is incurred by sampling only 50 people at a time from the population.
